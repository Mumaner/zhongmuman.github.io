gitee+hexo next主题blog搭建经验总结

  • 为什么想要blog
    • 就喜欢自己的东西

系统指引

  • 环境配置：[不再列举教程。注意node还需要在电脑属性-高级系统设置-环境变量中进行配置]
  • hexo博客搭建：史上最全的 Hexo 博客搭建配置完全指南

小技巧：直接输入node查看是否安装好node
注意：设置ssh时，是给账户设置，不是在单一的repository中设置

  • 进阶

    • 一台电脑多个账号：
      如何在一台电脑上使用多个git账号
      同一台电脑配置多个Git账户
      两者搭配食用
    • next主题优化：略

      走过的坑

  • 主题配置

  • markdown显示文章不全
    解决办法：重头再来（我的意思是blog也换个地方新建一个，除了环境不须配置一切重头再来）

  • hexo d提示错误
    解决办法： 重头再来

  • hexo d无法上传
    解决办法：删除文件夹[.deploy_git]

  • hexo d上传成功后网站无变化
    解决办法：手动更新page

  • 网站更新后，自己的电脑却没有更新
    可能是因为缓存原因。可以尝试禁用缓存（f12）

    blog常用操作

  • 打开git bash: 在正确的路径上右键打开

  • 新建post：

    hexo new “文章名称”

  • 新建page:

    hexo new page “page名称”

    注意要在index.md文件中，写入layout:[page名称]

  • 上传：

    hexo clean
    hexo g
    hexo s # 生成本地localhost:4000; ctrl+c关闭
    hexo d # 上传到repository

一些经验

  • 好看的主题

    • 我喜欢的（简约的）：
      chic♥：
      优点： 主页是全屏的图√； 正文居中，右侧有导航栏√
      放弃理由：1.我想要有背景图，且正文有一定的背景色不至于全透明。而chic我只能做到前者； 2.用户较少，网上教程很少
      next:
      优点： 教程多
      缺点： 侧边框的导航目录一出来，正文就不居中了
      我是用的：nest-mist
    • 热门：
      Sakura
      matery
      butterfly

  • 主题配置方法

    • 下载：
      法一：在github上直接clone-download zip [找到theme的方法：hexo theme官网]
      法二： 在git bash中直接输指令 [缺点：有时下载失败]
    • 我需要的配置：
      图标、头像、背景图片更换 title、 description 更换 pages： home about archieves categories tags 调整各框体透明

      教程：hexo下NexT的主题背景及框体透明度修改; Hexo+next的侧边栏背景与字体颜色设置方法

      文章加密：

      npm install –save hexo-blog-encrypt
      直接在markdown抬头输入password

      评论功能：畅言

      next已内置好
      教程：hexo评论_hexo博客之next主题添加畅言评论

  • 自定义页面

    • 新建： hexo new page “guestbook”，并在index文件中增添type行（我增加成了layout行）
    • 添加menu： 在主题下的_config中，仿造原menu格式进行添加guestbook[ps:图标：Font Awesome]
    • 为guestbook匹配对应中文“留言”：寻找其所在配置页的方法：直接在vscode中搜索hexo文件下所有含“首页”的子文件
    • index页面可以直接编辑

  • 小技巧

    • next主题下方心形图标的来源：Font Awesome
    • hexo有plugin(扩展)
    • 每次修改前都先备份
    • 在blog-source中新建[_data],下创建styles.styl,用于存储自己的设置
    • 了解一些常用名称含义有利于摸索[e.g.page sidebar footer body toc(导航目录)…]
    • 代码：# // 为注释

      待办

  • home; about 页面优化

  • 背景图片能否定时更换[参照：matery]

  • 复制正文时显示版权声明

  • archieves

Good luck to you!
